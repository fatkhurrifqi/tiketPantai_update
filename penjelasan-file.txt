------------------------------------------------------------------
FOLDER UTAMA (KONFIGURASI DAN DATA)
------------------------------------------------------------------

[ db.php ]
File ini adalah pintu masuk tunggal ke database, artinya seluruh file lain yang butuh akses data cukup memanggil file ini tanpa harus menulis ulang pengaturan koneksi. Di dalamnya dibuat koneksi ke MySQL menggunakan PDO dengan alamat host, nama database, username, dan password. Pengaturan PDO dipilih agar sistem langsung memberi peringatan ketika terjadi kesalahan, mengembalikan data dalam bentuk larik, dan memakai prepared statement asli sehingga aman dari serangan SQL injection. File ini mengembalikan variabel koneksi yang kemudian dipakai oleh semua halaman yang membutuhkan database.

[ database.sql ]
File ini berisi cetak biru atau struktur database sekaligus data awal. Di dalamnya dibuat enam tabel, yaitu users untuk akun pengguna dan admin, destinations untuk data pantai, ticket_types untuk jenis tiket dan fasilitas beserta harganya, orders untuk catatan pesanan, order_items untuk rincian tiket dalam setiap pesanan, dan reviews untuk ulasan pengunjung. Selain membuat tabel, file ini juga menyisipkan data contoh berupa satu akun admin, satu akun pengguna demo, tiga pantai yaitu Pantai Klothok, Pantai Sembukan, dan Karang Payung, serta beberapa jenis tiket untuk masing-masing pantai. File ini cukup diimport satu kali lewat phpMyAdmin atau MySQL untuk membangun seluruh struktur dan data dasar aplikasi.

[ seed.php ]
File ini adalah pembenih akun yang dijalankan lewat baris perintah, tujuannya membuat akun admin dan akun pengguna contoh secara cepat. Password yang dimasukkan langsung dienkripsi menggunakan fungsi hash bawaan PHP sehingga tidak tersimpan dalam bentuk teks biasa, dan proses penyimpanan memakai perintah yang tidak akan menimbulkan error walau email sudah pernah ada sebelumnya. Setelah dijalankan, file ini menampilkan konfirmasi email dan password akun yang bisa langsung dipakai untuk login.


------------------------------------------------------------------
FOLDER UTAMA (HALAMAN PUBLIK)
------------------------------------------------------------------

[ index.php ]
File ini adalah halaman beranda atau pintu depan aplikasi yang dilihat pertama kali oleh pengunjung. Isinya menampilkan slideshow foto pantai yang berjalan otomatis, lalu daftar kartu destinasi yang menampilkan gambar, lokasi, rating, dan harga tiket masing-masing pantai. Navbar di bagian atas berubah secara otomatis sesuai status login dan peran pengguna, misalnya menampilkan menu admin hanya untuk akun admin. Dari halaman ini pengunjung bisa mengklik sebuah pantai untuk melihat detailnya, atau langsung menekan tombol beli tiket untuk menuju halaman pemesanan.

[ destinations.php ]
File ini bukan halaman yang ditampilkan ke pengguna melainkan sebuah penyedia data dalam format JSON untuk keperluan integrasi dengan aplikasi lain seperti frontend Next.js. Ketika dipanggil, file ini mengambil seluruh data destinasi aktif beserta tiketnya dari database, kemudian mengubahnya menjadi format yang rapi dan konsisten agar mudah dibaca oleh program lain. Nama variabel yang diubah dari gaya ular menjadi gaya unta agar sesuai dengan kebiasaan penulisan di aplikasi JavaScript modern.

[ destination.php ]
File ini adalah halaman detail untuk satu pantai tertentu, yang dipanggil menggunakan parameter slug pantai di alamat. Di sini pengunjung melihat informasi lengkap tentang pantai tersebut seperti deskripsi, lokasi, jam buka, harga, dan kumpulan ulasan dari pengunjung lain lengkap dengan bintang penilaian. Halaman ini juga menyediakan form ulasan dengan bintang interaktif dari satu sampai lima dan kolom komentar, sehingga pengguna yang sudah login bisa memberi penilaian atau mengedit penilaian mereka sebelumnya. Notifikasi sukses atau gagal akan muncul kembali di halaman ini setelah pengguna mengirim ulasan.

[ checkout.php ]
File ini adalah halaman pemesanan tempat pengguna memilih jenis tiket, jumlah tiket, tanggal kunjungan, dan metode pembayaran. Pengguna bisa menambah atau mengurangi jumlah setiap jenis tiket, dan ringkasan total harga akan terus diperbarui secara langsung sebelum pesanan dikirim. Metode pembayaran dikelompokkan menjadi transfer bank, dompet digital, QRIS, dan bayar di lokasi, dan tombol kirim pesanan hanya aktif ketika sudah ada item yang dipilih dan metode pembayaran sudah lengkap. Seluruh hasil pilihan ini kemudian dikirim untuk diproses dan disimpan.

[ process_order.php ]
File ini menerima seluruh data pemesanan dari halaman checkout dan bertugas memvalidasi sekaligus menyimpan pesanan ke database. Harga setiap tiket diperiksa kembali berdasarkan data asli di database, bukan sekadar mempercayai angka yang dikirim dari sisi pengguna, sehingga harga tidak bisa dipalsukan. Sistem membuat nomor pesanan unik yang berisi kombinasi waktu dan kode acak, lalu menyimpan pesanan beserta seluruh rinciannya dalam satu transaksi yang utuh, artinya bila ada satu bagian yang gagal maka seluruh penyimpanan dibatalkan. Setelah berhasil, halaman sukses ditampilkan lengkap dengan ringkasan pesanan dan instruksi pembayaran sesuai metode yang dipilih.

[ orders.php ]
File ini menampilkan riwayat seluruh pesanan milik pengguna yang sedang login beserta statusnya, mulai dari menunggu pembayaran, sudah dibayar, selesai, hingga dibatalkan. Setiap pesanan menunjukkan nomor pesanan, destinasi, tanggal kunjungan, rincian tiket, total harga, dan status terbaru. Khusus untuk pesanan yang masih berstatus menunggu, halaman ini juga menampilkan kotak instruksi pembayaran seperti nomor rekening, gambar QRIS, atau tujuan dompet digital agar pengguna tahu ke mana harus membayar. Halaman ini hanya bisa diakses oleh pengguna yang sudah login.

[ orders_update.php ]
File ini adalah penyedia data untuk mengubah status sebuah pesanan, biasanya dipakai oleh halaman admin atau aplikasi lain. File ini menerima permintaan dengan metode khusus beserta nomor pesanan dan status baru, lalu memeriksa apakah status tersebut termasuk di antara nilai yang diizinkan yaitu menunggu, sudah dibayar, dibatalkan, atau selesai. Jika valid maka status di database diperbarui dan dikembalikan dalam bentuk JSON, sedangkan bila ada kesalahan file ini memberikan kode kesalahan yang sesuai agar pengirim tahu apa yang salah.

[ review_save.php ]
File ini menangani penyimpanan dan pembaruan ulasan dari pengguna terhadap sebuah destinasi, dengan aturan satu pengguna hanya boleh memberi satu ulasan untuk satu pantai. Ketika pengguna mengirim ulasan baru, sistem menambah hitungan ulasan dan menghitung rata-rata rating baru untuk pantai tersebut, lalu menyimpannya kembali ke data pantai agar tidak perlu dihitung ulang setiap kali halaman dibuka. Bila pengguna mengedit ulasan lama, sistem menyesuaikan rata-rata berdasarkan selisih bintang lama dan baru. Setelah selesai, pengguna dikembalikan ke halaman detail pantai beserta pemberitahuan hasilnya.

[ payments.php ]
File ini adalah pusat pengaturan seluruh metode pembayaran dan sama sekali tidak menyentuh database, fungsinya hanya menyimpan daftar pilihan pembayaran dalam satu tempat agar mudah diubah. Di dalamnya terdapat daftar empat kelompok pembayaran yaitu bank dengan beberapa nama bank, dompet digital dengan beberapa aplikasi populer, QRIS beserta lokasi gambar, dan opsi bayar di lokasi. File ini juga berisi fungsi penerjemah yang mengubah label dan kode pembayaran yang tersimpan di pesanan menjadi informasi yang mudah dibaca, misalnya mengubah kode bca menjadi Bank BCA lengkap dengan nomor rekening.


------------------------------------------------------------------
FOLDER auth (AUTENTIKASI)
------------------------------------------------------------------

[ auth/login.php ]
File ini menampilkan form login sekaligus memprosesnya. Pengguna memasukkan email dan password, lalu sistem mencari akun dengan email tersebut dan mencocokkan password menggunakan fungsi verifikasi hash yang aman. Bila cocok, data pengguna disimpan ke dalam sesi sehingga dianggap login di seluruh halaman, lalu pengguna diarahkan kembali ke beranda. Password tidak pernah disimpan atau dibandingkan dalam bentuk teks biasa karena seluruhnya menggunakan enkripsi satu arah. Bila gagal, pesan kesalahan ditampilkan agar pengguna tahu kalau email atau passwordnya tidak tepat.

[ auth/register.php ]
File ini menampilkan form pendaftaran akun baru sekaligus memprosesnya. Data yang diminta meliputi nama, email, nomor telepon, password, dan konfirmasi password, lalu semuanya divalidasi mulai dari kelengkapan, format email yang benar, kesamaan password dan konfirmasinya, panjang minimal password, hingga memastikan email belum pernah dipakai orang lain. Password langsung dienkripsi sebelum disimpan, akun dibuat dengan peran pengguna biasa, dan pengguna langsung dianggap login tanpa perlu masuk lagi. Bila ada yang tidak valid, pesan kesalahan ditampilkan kembali di form.

[ auth/logout.php ]
File ini berfungsi mengakhiri sesi login pengguna dengan membersihkan seluruh data sesi, menghapus cookie sesi, lalu menghancurkan sesi sepenuhnya. Tujuannya agar pengguna yang sudah keluar tidak bisa kembali mengakses halaman pribadi tanpa login ulang. Setelah selesai, pengguna dikembalikan ke halaman beranda. File ini pendek namun penting karena menjaga keamanan akun terutama saat perangkat dipakai bersama orang lain.


------------------------------------------------------------------
FOLDER admin (PANEL ADMIN)
------------------------------------------------------------------

[ admin/index.php ]
File ini adalah dashboard utama untuk admin, menjadi pusat kendali seluruh aktivitas sistem. Hanya pengguna dengan peran admin yang boleh masuk, sedangkan lainnya akan diarahkan ke halaman login. Bagian atas menampilkan ringkasan statistik seperti total pengguna, total pesanan, total pendapatan dari pesanan yang sudah dibayar atau selesai, dan jumlah destinasi aktif. Di bawahnya terdapat modul pengelolaan pesanan lengkap dengan menu ubah status, daftar pengguna, serta pengelolaan destinasi yang mencakup tambah, ubah, hapus, dan aktifkan atau nonaktifkan pantai. Beberapa fungsi JavaScript di halaman ini mengatur tampilan form, kotak dialog detail pesanan, dan konfirmasi sebelum menghapus data.

[ admin/stats.php ]
File ini adalah penyedia data statistik dalam format JSON untuk mengisi ringkasan dashboard. Ketika dipanggil, file ini menghitung total pengguna, total pesanan, total pendapatan, pesanan dikelompokkan berdasarkan status, dan daftar pesanan terbaru, lalu mengembalikannya dalam bentuk yang siap dipakai oleh tampilan dashboard atau aplikasi lain. File ini tidak menampilkan halaman langsung, melainkan berperan sebagai sumber data di latar belakang.

[ admin/users.php ]
File ini adalah penyedia data daftar seluruh pengguna dalam format JSON. Isinya menampilkan nama, email, dan peran setiap akun agar bisa ditampilkan dalam tabel di dashboard admin atau dipakai oleh aplikasi lain. Seperti file statistik sebelumnya, file ini berperan sebagai sumber data di latar belakang dan tidak menampilkan halaman tersendiri kepada pengguna.

[ admin/tickets.php ]
File ini adalah halaman untuk mengelola jenis tiket dan fasilitas yang tersedia di sebuah destinasi. Admin memilih satu pantai, lalu halaman menampilkan daftar tiket dan fasilitas beserta harganya, dilengkapi dengan tombol untuk menambah, mengubah, dan menghapus. Penambahan dan perubahan dilakukan melalui form dalam kotak dialog yang muncul di layar, sementara penghapusan meminta konfirmasi terlebih dahulu untuk menghindari kesalahan. Seluruh aksi ini kemudian dikirim ke file penyimpan khusus untuk diproses.

[ admin/destination_save.php ]
File ini memproses seluruh aksi pengelolaan destinasi, meliputi menambah, mengubah, menghapus, serta mengaktifkan atau menonaktifkan pantai. Nama pantai secara otomatis diubah menjadi slug yang ramah alamat web dan dipastikan unik, sedangkan unggahan gambar dilengkapi pemeriksaan ketat terhadap jenis dan isi asli file untuk mencegah file berbahaya menyamar sebagai gambar. Bila ada kesalahan tertentu, misalnya pantai tidak bisa dihapus karena masih dipakai pesanan, sistem memberi pesan yang mudah dipahami. Setelah selesai, admin diarahkan kembali ke dashboard dengan kode pemberitahuan hasil.

[ admin/ticket_save.php ]
File ini memproses seluruh aksi pengelolaan tiket dan fasilitas, meliputi menambah, mengubah, dan menghapus jenis tiket pada sebuah destinasi. File ini menerima data dari halaman kelola tiket, lalu menyimpan, memperbarui, atau menghapus baris tiket yang sesuai di database. Setelah selesai, admin diarahkan kembali ke halaman kelola tiket pantai yang dimaksud disertai kode pemberitahuan hasilnya. File ini menjaga agar daftar tiket selalu dapat diperbarui oleh admin tanpa perlu menyentuh struktur database secara langsung.


------------------------------------------------------------------
FOLDER assets (TAMPILAN DAN INTERAKSI)
------------------------------------------------------------------

[ assets/app.css ]
File ini berisi pengaturan tampilan tambahan di luar kerangka gaya utama, fokusnya membuat antarmuka terlihat rapi dan hidup. Di dalamnya terdapat palet warna khas yaitu perpaduan teal dan cyan, gaya bilah gulir yang disesuaikan, gradien berlapis untuk bagian hero, serta animasi muncul saat halaman digulir. Kartu destinasi dibuat terangkat dan gambarnya membesar saat disorot kursor, tombol diberi efek kilau, dan slideshow dilengkapi animasi gerak halus. Sebagai pertimbangan aksesibilitas, seluruh animasi akan dimatikan secara otomatis bila pengguna memilih untuk mengurangi gerakan di perangkatnya.

[ assets/app.js ]
File ini berisi kode interaksi global yang dipanggil di seluruh halaman publik. Ada tiga fungsi utama, yang pertama mengatur animasi munculnya elemen saat halaman digulir dengan fallback bila peramban tidak mendukungnya, yang kedua membuat navbar mendapatkan bayangan setelah pengguna menggulir ke bawah, dan yang ketiga menjalankan slideshow gambar latar di bagian hero secara otomatis disertai titik navigasi dan jeda otomatis ketika tab tidak sedang aktif. File ini menjaga pengalaman pengguna tetap mulus dan konsisten di mana pun mereka berada di aplikasi.

Folder ini membesar seiring banyaknya destinasi yang dikelola dan hanya berisi aset gambar, bukan kode program.

------------------------------------------------------------------
RINGKASAN ALUR UNTUK PENUTUP PRESENTASI
------------------------------------------------------------------

Aplikasi ini bekerja dengan alur yang sederhana dan runtut. Pengunjung masuk lewat halaman beranda untuk melihat daftar pantai, lalu memilih satu pantai untuk melihat detail dan ulasannya. Ketika pengunjung menekan tombol beli tiket, mereka diarahkan ke halaman checkout untuk memilih jenis dan jumlah tiket, tanggal kunjungan, dan metode pembayaran. Setelah pesanan dikirim, sistem menyimpannya secara aman ke database dan menampilkan halaman sukses beserta instruksi pembayaran. Pengguna dapat melihat seluruh riwayat pesanannya di halaman pesanan saya, sementara admin mengelola seluruh aktivitas lewat dashboard, mulai dari mengubah status pesanan hingga menambah dan mengatur destinasi serta tiketnya. Seluruh konektivitas data terpusat di file db.php, dan seluruh struktur tabel didefinisikan di database.sql, sehingga sistem tetap rapi, konsisten, dan mudah dikembangkan lebih lanjut.
