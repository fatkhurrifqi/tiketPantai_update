Tentang

Apa itu TiketPantai?

Platform pemesanan tiket masuk & sewa fasilitas pantai secara online — pesan dari rumah, nikmati pantainya tanpa antrian.

🎯 Tujuan

Digitalisasi penjualan tiket wisata Kecamatan Paranggupito agar lebih cepat & transparan.

👥 Pengguna

Wisatawan (beli tiket) & Admin (kelola destinasi, tiket, pesanan, status).

🏖️ Destinasi

Pantai Klothok, Pantai Sembukan, Karang Payung (+ dapat ditambah admin).

💳 Pembayaran

Transfer Bank, E-Wallet, QRIS, atau Bayar di Lokasi.

Teknologi

Stack & Tools

PHP 7.4+ (native) MySQL / MariaDB PDO Prepared Statement Session (PHP native) Tailwind CSS (CDN) Font Awesome Vanilla JavaScript bcrypt (password_hash) reveal.js (presentasi)

Backend

PHP murni tanpa framework — logika & view menyatu di file .php; DB via PDO.

Frontend

Tailwind utility class + app.css/app.js untuk animasi & micro-interactions.

Database

Entity Relationship (ERD)

users (1) ─────< orders >───── (1) destinations
                       │                    │
                       │                    └────< ticket_types
                       │                           ▲
                       └────< order_items >────┘

reviews ──> destinations   (1 user : 1 destinasi, unik)
reviews ──> users

Relasi: 1 pesanan punya banyak item; 1 destinasi punya banyak jenis tiket; 1 user hanya boleh 1 ulasan per destinasi.

Modul Inti

Konfigurasi & Database

🔧 db.php

Koneksi tunggal via PDO. Opsi: ERRMODE_EXCEPTION, FETCH_ASSOC, EMULATE_PREPARES=false (anti SQL injection). Di-return ke semua file.

🗄️ database.sql

6 tabel + seed (admin, user, 3 pantai, 7 tiket/pantai). FK dengan ON DELETE CASCADE.

🌱 seed.php

Buat akun demo via CLI (php seed.php). Password di-hash bcrypt dengan INSERT IGNORE.

💳 payments.php

Konfigurasi Bank, E-Wallet, QRIS, Lokasi. Fungsi resolve_payment() terjemahkan metode → instruksi.

Transaksi

Checkout & Pemesanan

🛒 checkout.php

Pilih jenis & jumlah tiket, tanggal kunjungan, metode pembayaran. JS menghitung total real-time & memvalidasi sebelum submit.

✅ process_order.php

Validasi harga dari DB (bukan client), generate TP-timestamp-xxx, lalu simpan via transaksi (orders + order_items).

💡 Keamanan: harga & total dihitung ulang di server, jadi client tidak bisa memanipulasi harga.

Keamanan

Autentikasi

🔐 login.php

Cocokkan email + password_verify() bcrypt → simpan $_SESSION['user'].

📝 register.php

Validasi (email unik, min 6 karakter, konfirmasi) → password_hash() → auto-login.

🚪 logout.php

Bersihkan session & cookie, lalu session_destroy().

🛡️ Role

user & admin. Halaman admin dicek role === 'admin'.

Back-office

Dashboard Admin

👥Pengguna 🛒Pesanan 💰Pendapatan 📍Destinasi

admin/index.php — statistik + tabel manajemen:

  • Pesanan — ubah status via dropdown + modal detail
  • Pengguna — daftar nama/email/role
  • Destinasi — CRUD lengkap dengan upload gambar

Back-office

CRUD Destinasi & Tiket

💾 destination_save.php

create/update/delete/toggle_active. Otomatis slugify() + slug unik + upload gambar dengan validasi ekstensi & MIME type.

🎫 tickets.php + ticket_save.php

Kelola jenis tiket/fasilitas per pantai (nama, harga, satuan, keterangan) — tiap pantai bisa berbeda.

⚠️ Proteksi: destinasi/tiket yang sudah dipakai pesanan tidak bisa dihapus (FK constraint 1451).

Quality

Aspek Keamanan

  • 🔐 SQL Injection — semua query pakai PDO Prepared Statement.
  • 🔑 Password — di-hash bcrypt (password_hash), tidak plaintext.
  • 🛡️ Authorization — guard login & role admin di setiap handler sensitif.
  • 🖼️ Upload — validasi ekstensi dan MIME type asli gambar.
  • 💸 Harga — dihitung ulang di server, bukan dari input client.
  • 🏷️ XSS — output di-escape dengan htmlspecialchars().
  • ↩️ Integritas — transaksi DB (commit/rollback) + FK cascade.

Hands-on

Cara Menjalankan

  1. Import database.sql via phpMyAdmin / MySQL CLI.
  2. Sesuaikan kredensial di db.php (host, nama DB, user, pass).
  3. (Opsional) Jalankan php seed.php untuk akun demo.
  4. Buka http://localhost/tiketPantai/ di Laragon.
